## Codebase Patterns
- Use Node.js route handlers and server-only modules for all pi SDK integration.
- Keep generated audio under `.data/audio` and expose it through `/api/audio/[id]` instead of direct file paths.
- Model/TTS configuration should come from env vars with safe defaults and friendly runtime errors.
- Configure `serverExternalPackages` for pi SDK packages so Next.js does not try to bundle Node-heavy agent dependencies.
- Create per-session workspaces under `.data/sessions/<sessionId>/workspace` and keep session records beside them for isolated pi conversations.
